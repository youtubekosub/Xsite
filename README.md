# 目的
次世代型アカウントサービス「仙人Account」をフルスクラッチで構築してください。
シンプルかつモダン、そして他アプリとの連携（SSO）を前提としたプロフェッショナルな仕様を目指します。

# 技術スタック
- Backend: Node.js (Express)
- Authentication: Passport.js (Google OAuth 2.0連携)
- Session: express-session (メモリ管理)
- Frontend: HTML5, CSS3 (Tailwind CSS 3.0 CDN), Vanilla JavaScript
- Security: API Key (UUID) 発行機能

# 必須機能とファイル構成（省略なしですべて出力すること）
1. server.js (バックエンド)
   - Google認証の実装
   - ログインユーザーへのユニークなAPIキー（アクセスキー）の自動割り当て
   - 他アプリからAPIキーを使用してユーザーデータ（名前、メール等）をJSONで取得できる外部連携用エンドポイント
2. index.html (ランディング兼ログイン画面)
   - 白を基調とした、清潔感と信頼感のあるGoogleスタイルのUI
   - 目立つ「Googleでログイン」ボタン
3. dashboard.html (ユーザー管理画面)
   - 自分のプロフィール、メールアドレス、そして「他アプリ連携用アクセスキー」を表示・管理する
   - アクセスキーの「コピー機能」と「再発行機能」を実装
4. admin.html (開発者パネル)
   - 他のアプリに埋め込むための「仙人ログインボタン」のHTML/CSSコードを生成・表示する機能
   - 外部からデータを取得するためのAPIドキュメントの記載

# UI/UXの要件
- コンセプト: 「明るい、使いやすい、かっこいい、プロ仕様」
- デザイン: 余白をたっぷり取り、丸みのあるカードデザイン（Rounded-3xl以上）、太いフォント、青（Blue-600）をアクセントカラーにする
- アニメーション: ページ遷移やボタンのホバー時の滑らかな挙動

# 出力形式
すべてのファイルを1つの回答内に、省略や「既存のロジックを流用」といった表現を使わず、完全にコピペで動作する状態で出力してください。
